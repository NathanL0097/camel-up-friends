# 好友桌游馆

一个支持多款独立游戏模块的在线好友桌游平台。平台统一负责游戏目录、创建房间、邀请链接、玩家身份、断线重连和实时动作分发；每款游戏分别实现服务端规则与浏览器界面。

目前接入两款游戏：

- “沙漠驼队竞速”私人技术演示，支持 2–8 人实时联机。
- 原创股票预测游戏《开盘！》第二版原型，支持 2–6 人掷骰选角、角色技能、上市事件、秘密下单、匿名事件揭示和8轮终局结算。

## 本地启动

需要 Node.js 20 或更新版本：

```bash
npm install
npm start
```

然后打开 <http://localhost:3000>。同一 Wi-Fi 下，可把 `localhost` 换成这台电脑的局域网 IP 后分享给朋友。

## 项目结构

```text
server.js                         HTTP 与 Socket.IO 接入层
src/platform/room-service.js     通用好友房、重连、开局与动作分发
src/games/index.js               服务端游戏注册表
src/games/camel-race/            骆驼竞速服务端规则模块
src/games/market-opening/        《开盘！》服务端规则模块
public/app.js                    平台大厅、身份、邀请和房间客户端
public/games/camel-race.js       按需加载的骆驼竞速界面与动画模块
public/games/market-opening.js   《开盘！》界面与动态股价图模块
```

首页通过 `/api/games` 读取游戏目录。创建房间后，房间会固定保存 `gameId`；服务端通过统一的 `game:action` 事件把操作分发给对应游戏模块。浏览器根据游戏元数据中的 `clientScript` 自动加载相应界面，因此新增游戏不需要复制好友房和重连逻辑。

## 新增一款游戏

1. 在 `src/games/<game-id>/` 实现规则、公开状态和动作处理器。
2. 在 `src/games/index.js` 注册游戏元数据与服务端模块。
3. 在 `public/games/<game-id>.js` 注册对应的客户端工厂，负责挂载游戏界面、规则说明和动画。
4. 为规则与平台动作分发补充自动测试。

## 已实现的平台能力

- 可扩展游戏目录与游戏元数据接口
- 6 位好友房码和邀请链接
- 无账号游客加入与浏览器本地身份重连
- 通用创建、加入、开局、重新开局和动作分发
- 旧版骆驼房间事件的兼容入口
- 每位玩家独立的私密公开状态
- 手机和电脑响应式界面

## 公开部署

这是一个普通 Node.js Web 服务，可部署到 Render、Railway、Fly.io 等支持 WebSocket 的平台。启动命令为 `npm start`，平台需提供 `PORT` 环境变量。

当前房间仍保存在单台服务器内存中，服务器重启后会消失。正式长期运营前需要把房间、账号和权益迁移至 Redis 与数据库。
